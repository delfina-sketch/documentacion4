# 💸 Promociones

{% hint style="info" %}
Desde Pow aconsejamos ante la habilitación del módulo para crear Promociones, el exhaustivo, minucioso y periódico control de este bloque dado que la creación de una Promoción es estar creando dinero para utilizar en la web. Asimismo recomendamos la revisión de los perfiles y sus roles asignados para evitar y/o disminuir acciones que comprometan la integridad del sitio tanto como acciones mal intencionadas de terceros.

Lo aconsejable en este punto es que **solo un grupo muy reducido de perfiles** cuenten con acceso a este módulo.
{% endhint %}

### Promociones <a href="#promociones" id="promociones"></a>

Este módulo es administrable desde la sección con su mismo nombre, **Promociones.**

### Crear promoción <a href="#crear-promocion" id="crear-promocion"></a>

Allí, haremos click sobre "**Crear promoción**", donde tendremos las siguientes opciones:

<figure><img src="https://pow-ecommerce.gitbook.io/vtex-clientes/~gitbook/image?url=https%3A%2F%2Flh7-us.googleusercontent.com%2Fny3aFQind90SoVIPRvUqo6FmmRl8RIbmSwY_aX7UStdh4nV29NrjsbTfKUwTYpzauT4jCZBhkSaColugbbkm9X7TVInPeQGTl4wQOc2r37cc7CgWoskgwx6e6IpOXLKdOeS2H23UV1xh17U2tJnY67s&#x26;width=768&#x26;dpr=4&#x26;quality=100&#x26;sign=a5b6bfe4&#x26;sv=2" alt=""><figcaption></figcaption></figure>

* **Regular:** Es el más utilizado, permite asignar condiciones y brindar cierto descuento porcentual o numérico.
* **Comprar juntos:** Permite aplicar un descuento cuando se tienen ciertos SKUs en el mismo carrito.
* **Más por menos:** Se utiliza para promociones del tipo "lleve 3, pague 2".
* **Descuento progresivo:** Es utilizado para descuentos progresivos, del estilo "Lleve 2 y tenga 10% de descuento, lleve 4 y tenga un 5% extra (total 15%)"
* **Compra uno y lleva uno gratis:** Sirve para dar cierto producto o envío gratis a quien compre determinado SKU.
* **Promoción de campaña:** Son promociones vinculadas a un público de campaña objetivo, que debe ser configurado previamente.

### Promoción regular <a href="#promocion-regular" id="promocion-regular"></a>

Le damos click a **Crear promoción** y elegimos **Regular**, ya que es la promoción que comúnmente se usa.

*   **General:**

    * **Nombre:** Es el nombre que le daremos a la promoción para distinguirla de las demás promociones.
    * **Descripción:** Se puede agregar, además, una descripción de la promoción para que otras personas que ingresen puedan consultar de qué se trata la promoción.
    * **Fecha de inicio:** Se debe asignar fecha de inicio y de fin. En caso de querer hacerla por tiempo indefinido, pueden ponerse años lejanos, por ejemplo 2030.
    * **Hora inicio y fin:** Se debe asignar hora de inicio y de fin.
    * **Utilizar configuración de recurrencia:** Permite que la promoción se active únicamente en días específicos u horarios específicos recurrentes. Ejemplo, que se active sólo los lunes y martes, o que se active de 17 a 18.

* **Origen del pedido:** Se debe elegir entre las opciones: **Mi tienda (marketplace) o Tiendas de terceros (fulfillment)**
* **Políticas comerciales:** Se puede elegir para que aplique a Todas o sumar un condicional para incluir o excluir alguna.
* **Sellers:** Se puede elegir para que aplique a Todas o sumar un condicional para incluir o excluir alguno.

<figure><img src="https://pow-ecommerce.gitbook.io/vtex-clientes/~gitbook/image?url=https%3A%2F%2F3709566463-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F72B6lE7Nn0jKtcyB10gI%252Fuploads%252Fz7jsV7CqsI7FSBxcIxV8%252Fimage.png%3Falt%3Dmedia%26token%3D5295dd23-abbd-427d-8c35-3bc9d7d1d431&#x26;width=768&#x26;dpr=4&#x26;quality=100&#x26;sign=63cdc7b&#x26;sv=2" alt=""><figcaption></figcaption></figure>

*   **Regular:**

    *   **Si el cliente compra:** Se deben configurar los SKUs a los que aplica la promoción. Esta configuración se puede realizar para que aplique a todos los items de la tienda o también se puede configurar por SKUs específicos, Productos, Colecciones, Categorías o Marcas.

    * **El cliente se lleva:** Se debe configurar el beneficio que recibirá el cliente para esta promoción. Se puede elegir entre Descuento, Envío, Precio máximo, Regalo, Valor de fidelidad o Tabla de precios promocionales.&#x20;

Al ingresar por alguna de las opciones, nos mostrará las opciones para cada caso:

<figure><img src="https://pow-ecommerce.gitbook.io/vtex-clientes/~gitbook/image?url=https%3A%2F%2F3709566463-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F72B6lE7Nn0jKtcyB10gI%252Fuploads%252F9b4NsE0RvDyIHYFzbFOv%252Fimage.png%3Falt%3Dmedia%26token%3D9484c83d-debf-4ad2-ab1a-e12885c5246e&#x26;width=768&#x26;dpr=4&#x26;quality=100&#x26;sign=b85de05&#x26;sv=2" alt=""><figcaption></figcaption></figure>

*   **Condiciones:** Se deben definir las condiciones para aplicar dicha promoción. Se puede elegir entre Afiliados, Clúster de clientes, Medio de pago, Número de BIN de la tarjeta, Número de cuotas, Parámetro de UTM, Pedido de suscripción, Precio de lista, Precio del item, Sólo en la primera compra, Tags de marketing, Tipo de envío, Ubicación del envío, UTMI o Valor del pedido.

*   **Acumulación y competencia:** Permite definir los comportamientos de acumulación y competencia de la promoción. Se puede elegir entre: Permitir acumular con otras promociones o Permitir acumular con precios manuales.
